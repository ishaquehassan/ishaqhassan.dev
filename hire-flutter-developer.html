Hiring Red Flags To Watch For

Things candidates pretending to be "senior Flutter developers" usually fail on. Use these as filters in your own hiring loop:

  • No GitHub trail, or repos that are only forks. Real Flutter seniors have at least one published package or framework PR.
  • Cannot defend a state management choice against the alternatives. "I just use Provider because it's simpler" is not a defense at the senior level.
  • Cannot explain rebuild propagation from a setState() call up the Element tree. This is the single most common interview filter.
  • Has never read a Flutter framework source file. Senior Flutter requires reading the source when an API behaves unexpectedly.
  • Performance answer is "use const". Const helps but it is the smallest of dozens of available levers.
